# 2021-02-09 折腾智能电视软件安装

Sharp电视要装芒果Tv，没带U盘，用Http-server开启一个http服务器，通过在电视浏览器访问同个局域网下的计算机，下载apk，不料电视提示没有sd卡，还有没有 方法呢。
现在智能电视是android系统，有一个adb技术，可以通过电脑安装apk。于是开始漫漫的开启adb之旅，首先进入Sharp的工厂模式。断电情况下按着音量-和 信源键（不放），按电源键，电视上出现K模式，此时松开音量-和信源键，再同时按住音量-和频道-进入直到进入工厂模式，松开。但是却怎么也没找到adb选项。 作罢。（如果找到adb，下面使用adb调试工具，在电脑cmd下使用adb connect 电视在局域网下的ip， 然后安装apk）。
最终，还是找了个U盘，从u盘安装。
小插曲

  • 打电话咨询Sharp客服怎么打开adb，无果。
  • 从u盘安装apk，提示无法解析安装包，发现下载的apk适用android系统版本高于电视的android系统版本。结果下了低版本的apk。
  • apk安装不进去，发现存储总共2g，还剩200多M，当贝清理软件也无法清理，只得恢复到出厂设置，才Ok.

# 2021-02-23

  • 按boss需求，帮用户清理测试数据，qq远程使用命令行操作，首先思考的是软删除还是delete，选择软删除。
update xx set deleted = '1' where insert_time < '2021-01-20' and deleted = '0';

没过多久，用户反馈，数据不见了，原来那些演示数据直接被用户更新为正式使用数据。庆幸只是软删除，于是帮用户恢复，遇到一个问题，就是在insert_time <'2021-01-20'and deleted= '1' 的记录不只刚刚删除的这些，之前软删除的数据也在里面。无奈只好找出当初这批测试数据维护的时间区间进行删除

update xx set deleted = '0' where insert_time < '2021-01-20' and insert_time > '2021-01-16' and deleted = '1';

实际上如果之前删除的时候加上update_by作为标记，将更好恢复数据。

此外如果真的使用了delete，那么使用备份进行恢复。

  • 寻求帮助 看Spring官网如下红色框里的一段话，并实际动手，并未发现ApplicationContext里有getBeanFactory方法，于是在训练营群里进行提问。 最终在大家帮助下，发现原来忽略了这段话前面的重点，也就是在ApplicationContext的实现中。也就是这边的GenericApplicationContext。

# 2020-02-25 一段代码的简单思考

这个for语句前面是否需要判断list的size 结论：不需要 原因: 代码更简洁，忽略可能的效率提升。 分析：从字节码角度看，if这一行判断字节码指令确实比for要少些，因为for要取iterator和hasNext，而if只需取size。 这里我觉得真的要谈性能估计还要分List是ArrayList还是LinkedList，这才能判断hasNext和size谁快。所以权衡下忽略效率。就不需要if。

public class Test {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
        list.add("1");
        if(list.size() > 0) {
            for (String a : list) {
                System.out.println(a);
            }
        }
    }
}
