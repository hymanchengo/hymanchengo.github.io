# 一起学java之总览

# 1. java是什么

  • 一门面向对象的高级编程语言。
  • 技术平台
    • JDK（Java Development Kit）：Java开发工具包，包含Java程序设计语言，工具及工具API，Java运行时环境
      • JRE（Java Runtime Environment）: Java运行时环境，包含JVM，Java类库API
        • JVM（Java Virtual Machine）：Java虚拟机，分为客户端虚拟机和服务端虚拟机
          引自Java Platform Standard Edition 8 Documentation
    • Class文件格式
    • 来自商业机构和开源社区的第三方类库API
  • 技术应用领域划分
    • Java Card
    • Java SE
    • Java EE
    • Java ME

# 2. java语言特点

java语言的诞生出于两点

  • 时代发展必然，顺应大的软件环境
  • 语言发展必然，Java作为C，C++的后继语言，去除了C和C++中指针管理等一系列复杂性，简化了开发

简单、面向对象、分布式、多线程、动态、平台无关、可移植、高性能、健壮、安全
这些特性的描述可参照白皮书Java语言环境 (opens new window)

# 3. java版本历史

暂统计到Java SE 8，从JDK 1.3开始版本大约每两年发布一次，从JDK 9开始，版本每半年发布一次。

J2SE 1.2到J2SE 5.0版本名称由JDK变为J2SE，用于区分J2EE及J2ME。
从J2SE 5.0开始，原先的1.5作为内部版本号使用。此次版本号变更是为了更好反映J2SE的成熟度，稳定性，可扩展性和安全性。
从Java SE 6开始，版本号名称从J2SE变更为Java SE，而且从版本号中去掉.0。开发者内部版本号依然维持1.6.0。

# 语言发展简介

软件是人们和计算机沟通的媒介，如用音乐软件让计算机播放音乐。而软件是由程序员事先在电脑上编写并打包出在目标计算机上可运行的实体。 正如我们日常生活沟通需要语言一样，程序员和计算机沟通也需要一门语言，这们语言最初也最低级的就是机器语言，这是计算机唯一认识的语言。 该语言对计算机极为友好：它就是0和1组成的串。如010010111001。但让人以计算机的思维方式来理解01串就太为难人了。这01串代表啥含义呢， 即便是智慧过人的程序员也不能脱口而出，它既可能表示一条计算机指令，也可表示操作数的地址。让程序员去记住这些指令的二进制编码或者操作 内存地址，这显然效率极低且容易出错。程序员疲于应付程序的完成和正确性，创新更是无从谈起。好在懒惰是程序界的一大生产力，后来人们发明 了汇编语言，汇编语言实际上是机器码的助记符。但使用汇编语言的程序员日志显然好过多了。毕竟ADD指令可以用来替代书写它对应的00000011串。 再后来，出现了高级语言，这种语言更加符合人类的思维方式，就像使用自然语言一样， 如Java代码段：if(todayIsFriday) { System.out.println("i am happy！"); }是不是很好理解:如果今天是星期我，输出我很开心。 高级语言，主流的有面向过程的C和面向对象的Java、C#、C++等。当然JavaScript，Python，Go等近年来发展也很快，其实编程语言一路走来， 远不止我们熟悉或听过的那些，有很多语言也很优秀，只是没有被广泛使用。另外，经常看到一些帖子非要争个语言优劣，其实没意义，各语言有 适用的场景。如果觉得好，多学几门也是很轻松的，尤其是在掌握了一门语言之后。从语言发展的历程说一些我觉得其中体现的思想：从最初的机器 语言到我们所熟悉的高级语言，这里面体现了一种抽象（将具体的操作码用助记符表示）、封装及隔离变化（编译器，虚拟机等）及模块化（提供各类API）。高级语言中几乎看不到机器语言的身影。 能做到这点也得益于强大的编译器，脏活累活都被它干了。它将高级语言翻译为机器语言，对程序员是透明的(不可见的)。高级语言编写的程序的移植性 较机器语言要好，因为高级语言编写的代码不假定特定的机器。借助虚拟机甚至可以实现跨平台运行。程序员使用了高级语言编写程序基本是和底层解耦的， 特别是在特定领域出现流行的框架后，程序员更是可以集中精力开发业务特定问题。
