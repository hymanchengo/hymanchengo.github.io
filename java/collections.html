# Java collections framework

# 简介

# 集合

集合是表示一组对象的对象，有时称为容器。用于将一组元素放到单一单元中。用于CRUD内存数据。集合的概念我们并不陌生， 如火柴盒里的火柴就构成了这么一个集合，类似的还有一副扑克。

# 集合框架

集合框架是表示和操作各类集合的统一体系结构，独立于集合的实现细节。 它包含

  • 接口 表示集合的抽象数据类型，接口使得我们可以独立于具体的集合实现来操作集合。面向对象语言中，接口通常形成层次结构
  • 实现 集合接口的具体实现。本质上是可重用的数据结构
  • 算法 对实现集合的对象执行有用计算的方法，这些方法可以说成多态的，因为它可以作用在不同的集合实现上。本质上算法是可重用的功能 接口，实现，汇总操作，算法构成了java集合框架

集合框架提供了一套统一的接口完成不同场景下的数据计算，我们可以根据不同场景，选用合适的集合实现，调用集合框架提供的计算方法(算法)，完成对数据的处理需求

# 集合框架的优点

如下的API可以理解为一个函数

  • 降低使用门槛，你不需要自己编写数据结构和算法。也不需要写适配器对象或者转换代码来使不相关的API实现互操作性。集合框架都为你实现了。
  • 提升程序速度和质量，集合框架提供高性能的数据结构和算法。每个接口的不同实现可以切换。更多的时间被用于提示程序的质量和性能
  • 不同API之间可以通过传递集合参数来实现互操作性。
  • 降低学习和使用新API所花费的努力。以往API之间传递集合会用子API来操作对应集合。现在集合统一了，无需这些API。
  • 和上面优势类似，创建依赖集合的API时不需要重新造轮子，直接使用标准集合接口
  • 促进软件重用。遵循标准集合接口的新的数据结构和操作实现标准接口对象的新的算法同样可用于已有代码。

# LinkedList

  • 底层数据结构 双向链表，存储的元素在内存空间是非连续的。元素由数据及地址部分组成，元素间使用指针和地址连接。
  • 非线程安全
  • 特性 插入和删除速度快，随机方法速度慢。
  • 实现AbstractList和Deque(double-ended queue readed as deck) Deque接口既可以用作队列，也可以用作栈。分别有2套API来操作对应形态的数据结构
  • 优点 可以关联零散的内存空间，提升内存利用率。

# 数组和集合间的相互转换

数组和集合都是存储元素的容器。他们之间可以进行转换。

  • 数组转换为集合 Arrays.asList 使用Arrays.asList转换后的list结构是不能变更的（即不能添加和删除）。Arrays.asList仅仅将数组包装，让他看起来像个list

如下将数组转换为List后，可以操作List修改原数组的值，但是不能使用我们常使用的ArrayList的add,remove方法。一旦使用就会扔出UnsupportedOperationException。 通过源码可以看出asList返回的ArrayList类是Arrays工具类的内部静态类，它同样继承了AbstractList。遗憾的是它并没有重写AbstractList中的add和remove方法。 而AbstractList的add和remove实现默认扔出UnsupportedOperationException实现。除非子类对其进行复写。

Arrays.asList体现的是适配器模式，可以像List那样通过get和set获取和修改元素。 问题来了，为什么内部静态类ArrayList不实现add和remove方法呢。我认为这是设计初衷。这种设计只提供原数组的视图。数组本身具有固定大小。因此便不提供add和remove方法。

public class Arrays {
// ...
private static class ArrayList<E> extends AbstractList<E>
        implements RandomAccess, java.io.Serializable {
}
// ...
}
