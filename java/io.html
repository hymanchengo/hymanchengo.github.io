# IO

# 字节流示例

字节流操作是较为底层的操作，仅用于最原始的I/O。除此外应避免这种底层I/O操作。

public class CopyBytes {
    public static void main(String[] args) throws IOException {
        FileInputStream in = null;
        FileOutputStream out = null;

        try {
//            in = new FileInputStream("C:\\Users\\Hyman\\Desktop\\idoc\\xanadu.txt");
//            in = new FileInputStream("C:\\Users\\Hyman\\Desktop\\idoc\\outagain.txt");
            in = new FileInputStream("C:\\Users\\Hyman\\Desktop\\idoc\\1.mp4");
            out = new FileOutputStream("C:\\Users\\Hyman\\Desktop\\idoc\\2.mp4");
           
// int使用低8位值来保存字节值
            int c;

            while((c = in.read()) != -1) {
                out.write(c);
            }
        } finally { //finally块确保流的最终关闭，避免严重的资源泄露
            //当出现异常后，in仍保持null，因此这边在关闭流前做下判断。
            if(in != null) {
                in.close();
            }
            if(out != null) {
                out.close();
            }
        }
    }
}

# 字符流

java使用Unicode存储字符值。字符流I/O自动将内部格式转换为本地字符集或从本地字符集转换到内部格式。西方地区，本地字符集通常是 8位的ascii码超集。使用字符流代替字节流的程序自动适配本地字符集，并可以进行国际化。 如果字符集不是很优先的事，则可以简单使用字符流，不用关心字符集的问题。等到关心国际化时，再去关注及做微调。

所有字符流类均来自Reader和Writer。专用在文件I/O的字符流类是：FileReader和FileWriter 字符流是reader和writer的

  • 逐个字符为单元
public class CopyCharacters {
    public static void main(String[] args) throws IOException {
        FileReader inputStream = null;
        FileWriter outputStream = null;

        try {
            inputStream = new FileReader(ConstantValue.path + "xanadu.txt");
            outputStream = new FileWriter(ConstantValue.path + "characteroutput.txt");

// not work
//            inputStream = new FileReader(ConstantValue.path + "1.mp4");
//            outputStream = new FileWriter(ConstantValue.path + "characteroutput2.mp4");

//int使用低16位来保存字符值
            int c;
            while((c = inputStream.read()) != -1) {
                outputStream.write(c);
            }
        } finally {
            if(inputStream != null) {
                inputStream.close();
            }
            if(outputStream != null) {
                outputStream.close();
            }
        }
    }
}

字符流操作文件I/O和字节流的区别在于，字符流使用FileReader，FileWriter，而字节流使用FileInputStream,FileOutputStream。 字符流是字节流的包装形态。在内部实现上，字符流使用字节流来操作物理I/O，而字符流处理字符和字节间的翻译。FileReader使用 FileInputStream，FileWriter使用FileOutputStream。

有两种通用的字节到字符的桥接流：InputStreamReader和OutputStreamWriter。如果包内没有满足需求的字符流类，那么使用它们来 创建字符流。

字符IO通常以比单个字符更大的单元出现。一个常用的单元是行：一串末尾以行终结符结尾的字符串。行终结符回车换行\r\n，单个回车\r 或单个换行\r。支持所有行终结符使得程序可以读取任何广泛使用的操作系统上创建的文本文件。

  • 逐行为单元
public class CopyLines {
    public static void main(String[] args) throws IOException {
        BufferedReader inputStream = null;
        PrintWriter outputStream = null;

        try {
            inputStream = new BufferedReader(new FileReader(ConstantValue.path + "xanadu.txt"));
            outputStream = new PrintWriter(new FileWriter(ConstantValue.path + "characterlineoutput.txt"));

            String l;
            //返回一行
            while((l = inputStream.readLine()) != null) {
                //输出一行，这边的行终结符可能和输入文件中的行终结符不同
                outputStream.println(l);
            }
        } finally {
            if(inputStream != null) {
                inputStream.close();
            }
            if(outputStream != null) {
                outputStream.close();
            }
        }
    }
}

# 缓冲流

字节流和字符流的每次读取或写入是直接由底层OS处理的。这样程序较为低效。因为每次请求将触发磁盘访问，网络活动，或其他相当耗时的操作。 java平台实现缓冲I/O流来减少此类开销。缓冲输入流从一段内存（缓冲）中读取数据。当缓冲中空了时会调用底层输入API（来填充缓冲）。类似，缓冲输出流 将数据写入缓冲，当缓冲写满了，调用底层输出API。

程序可以将非缓冲流转换为缓冲流，使用包装技巧。未缓冲流对象传入缓冲流类构造函数的参数，如

inputStream = new BufferedReader(new FileReader("xanadu.txt"));
outputStream = new BufferedWriter(new FileWriter("characteroutput.txt"));

有四种缓冲流类用来包装非缓冲流：BufferedInputStream和BufferedOutputStream用来创建缓冲字节流，BufferedReader和BufferedWriter 创建缓冲字符流。

  • 刷新缓冲流 通常在关键点将缓冲写出去，而不等到缓冲填满是有意义的。这也就是刷新缓冲。 一些缓冲流支持自动刷新，通过制定可选的构造函数参数。当启用自动刷新，特定事件将触发缓冲刷新。比如自动刷新的PrintWriter对象 在每次调用println或format时将刷新缓冲。 通过调用flush方法可以手动刷新流，任何输出流都可以调用此方法，但只有缓冲流的调用才会有效果。
