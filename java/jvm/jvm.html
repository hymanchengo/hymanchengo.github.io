# java源代码

public class HelloWorld {
    public static void main(String[] args) {
        HelloWorld helloWorld = new HelloWorld();
    }
}

# 编译HelloWorld.java生成HelloWorld.class

javac HelloWorld.java

# 反汇编HelloWorld.class

javap -c HelloWorld.class通过-c命令查看反编译代码，如组成字节码的指令 字节码文件中包含了默认构造函数和main函数两块。默认构造函数调用了父类Object的构造函数。main函数中调用默认构造函数创建HelloWorld对象。

# 循环字节码

int循环示例代码片段

void spin() {
  int i = 0;
  for (i = 0; i < 100; i++) {;
  }
}

0 : iconst_0
1 : istore_1
2 : iconst_0
3 : istore_1
4 : iload_1
5 : bipush 100
7 : if_icmpge 16
10 : iinc 1,1
13 : goto 4
16 : return
  • 解析
    • 用了16个字节码
    • 第1个字节码iconst_0(0x3)，将int常量0压到操作数栈上
    • 第2个字节码istore_1(0x3c),将操作数栈的栈顶元素弹出，将其设置到局部变量表数组的第二个（索引1）位置处。
    • 第1个和第2个字节码实际上完成的是int i = 0;
    • 第3个第4个字节码完成的是for循环里的i = 0
    • 第5个字节码iload_1(0x1b)将局部变量表里索引为1（第二个位置）处的值压倒操作数栈
    • 第6个字节码bipush(0x10)将紧接着的字节符号扩展为int值，压入操作数栈
    • 第7个字节表示操作数100
    • 第8个字节比较弹出的100和栈顶i比较
    • 第9，10个字节表示指令地址，在if_icmp条件指令后的2个字节表示有符号的16位数表示的偏移量，执行从比较指令地址处跳到偏移量处执行
    • 第11个字节iinc表示如果第8个字节的比较条件不成立，那么增加索引为1处的局部变量表的值加上常量
    • 第12个字节1位索引为1处的局部变量表的值
    • 第13个字节1表示常量，其首先符号扩展为int
    • 第14个字节 goto表示(0xa7)无条件分支跳转
    • 第15(byte1),16(byte2)两个无符号字节构造(byte1 << 8 | byte2)一个有符号的16位偏移量。
    • 第16个字节return从方法返回

double循环示例代码片段

void dspin() {
  double i;
  for (i = 0.0; i < 100.0; i++) {;
  }
}

其反汇编的字节码为，//后面的为解释或者是Javap生成的注释

#2 = Double             100.0d

0 : dconst_0   //压入0.0
1 : dstore_1   //将0.0存储到局部变量表的1，2位置(double占2个位置，使用较小索引来访问)
2 : dload_1    //将局部变量表1，2位置的值压入栈
3 : ldc2_w  #2 // double 100.0d   //将常量池第二个字节开始的2个字节值，这边是100.0压入栈
6 : dcmpg      //比较从弹出的栈顶的2个元素 0.0 和100.0d 0.0 < 100.0d返回-1
7 : ifge 17    //将上一步的栈顶的结果-1和0比较 如果-1>=0走17，显然不成立，继续第10个字节码指令执行
10 : dload_1   //将局部变量表1位置的值0.0压入栈
11 : dconst_1  //将1.0压入栈
12 : dadd      //将栈顶的2个元素0.0, 1.0弹出并相加。结果1.0d入栈
13 : dstore_1  //将栈顶元素1.0存储到局部变量表1位置
14 : goto 2    //转到第2条指令
17 : return    //完成返回void
  • ldc2_w指令
    • 从运行时常量池压入操作数栈（宽索引）
    • 格式 ldc2_w indexbyte1 indexbyte2
    • 指令码(0x14)
    • 描述
      • 无符号的indexbyte1和indexbyte2装配成无符号的16位的索引指向当前类的运行时常量池 索引值是(indexbyte1 << 8 | indexbyte2)计算出。索引必须是当前类运行池的合法索引。该索引 对应的运行时常量池的条目必须是long或double类型的常量。对应的运行时常量值以long或double类型入栈。
    • 注意 只有ldc2_w指令的宽索引版本存在，不存在使用单字节索引压入long或double的ldc2指令。 ldc2_w指令只能用于压入double值集指定的double类型值，因为常量池中的double类型常量必须来自double值集

short循环代码片段

void sspin() {
  short i;
  for (i = 0; i < 100; i++) {;
  }
}

对应的反汇编指令

0 : iconst_0
1 : istore_1
2 : iload_1   //short被视为int
3 : bipush 100
5 : if_icmpge 16
8 : iload_1
9 : iconst_1
10 : iadd
11 : i2s    //这边是使用i2s指令将int截断为short。这是使用short,byte,char等类型时的额外开销  
12 : istore_1
13 : goto 2
16 : return

从上面int循环示例和double循环示例，可以看出，jvm对数值类型int的支持比double要好，对int提供了iinc及if_icmpge，而在double中并没有提供类似的指令，
double的示例中通过两条以上的指令来实现这样的效果，这主要原因处于jvm使用单字节来紧凑表示字节码的原因，因此jvm指令对于数据类型类型并不是正交的，
int类型上的部分操作，在其他类型中并不存在，而对于short,char,byte等的运算操作更多的是转换为int类型的相关指令操作。jvm偏爱int的主要原因是期望操作
数栈和局部变量表的高效实现。也是由于int在大多数程序中使用频率更加高。

总的来说，jvm对int直接支持，对short,byte,char缺少直接支持(转换为int类型操作)，对long及浮点型的支持次于int,主要缺少根据比较操作数的结果直接转移的单条指令

# 四则运算

int align2grain(int i, int grain) {
  return ((i + grain - 1) & ~ (grain - 1));
}

反汇编指令

0 : iload_1        // 从局部变量表1位置压入栈
1 : iload_2        // 从局部变量表2位置压入栈   
2 : iadd           // 弹出栈顶2个元素，相加结果压入栈 i + grain
3 : iconst_1       // 将常量1压入栈
4 : isub           // 弹出栈顶2个元素，相减结果压入栈，栈顶为 i + grain -1
5 : iload_2        // 将局部变量表2位置压入栈 grain
6 : iconst_1       // 将常量1压入栈
7 : isub           // grain - 1
8 : iconst_m1      // 将常量-1压入栈
9 : ixor           // 将栈顶2个元素弹出做异或运算，结果压入栈。基于~ (grain -1) = -1 ^ (grain - 1)，因此这边
10 : iand          // 将栈顶2个元素（分别是 i + grain - 1 和 ~ (grain -1)）做按位与运算，结果入栈
11 : ireturn       // 返回int
