# 面向对象编程基础

  • 面向对象编程（Object Oriented Programming）

  • 面向对象不是Java首创，之前的C++已采用面向对象的技术，Java采用面向对象也是考虑了网络应用等发展的趋势。Java吸收了C、C++中 大家已经熟悉的语法并针对它们复杂的特性方面进行简化，比如提供自动垃圾回收机制以取代手动申请释放内存，去掉C++中的头文件等。 因此Java较C、C++是简单的，对从C、C++转过来的程序员也是友好的。

  • 面向对象编程实际上是编程范式的一种，之前的C语言是面向过程的代表，另外还有一种函数式编程。

  • 面向对象编程的四大特性

    • 封装性 通过类封装数据和行为，实现信息隐藏和模块化
    • 继承性 通过类的继承实现代码的复用。
    • 多态性 程序的行为取决于实际接收消息的对象的行为
    • 动态绑定 编写程序时不必知道具体的类型。在运行期传入具体类对象。
      面向对象编程简单说来：我们需要编写职责划分的类。然后通过创建类的对象及向对象发送消息（调用对象的方法）来完成 程序的功能

  • 面向对象和面向过程有啥区别
    一个容易被忽略的点：面向过程在一个方法X中调用方法Y，那么方法Y的生命周期局限于方法X中。而面向对象中在X对象里创建的 Y对象的生命周期可以超出X对象，可以通过网络传递，存储到数据库中，也可以再还原出来。

# 一切皆对象

  • Java除了基本数据类型等少数特例外，任何表示都是以对象的形式存在，甚至基本数据类型也有对应的包装类。
  • 对现实世界进行建模，对象即模型。
  • 可以将对象想象成一个细胞，细胞核是对象状态，而细胞核和外界的交互都要通过细胞壁（对象的行为方法）

# 类

数据和行为的模板，从类中可以创建多个具有独立状态的对象

  • 实例方法定义对象可拥有的行为

# 构造方法

构造方法顾名思义，是在构造实例（对象）时调用以初始化实例的，它没有返回值，访问修饰符的作用同样适用于构造方法，默认是包级访问（后面会谈到访问修饰符）。

  • 无参构造方法
    Bird类有个默认的公共无参构造方法，我们不添加任何构造方法时，编译器会自动添加并调用该构造方法
  • 创建Bird实例
    我们可以使用new关键字来创建Bird对象Bird bird = new Bird()。Bird变量bird指向堆上创建好的Bird对象
  • 带参构造方法
    当我们显式定义了构造方法后，编译器将不会再为我们提供默认构造方法。如果提供了带参构造方法的同时，仍需要支持以无参的方式构造对象，则需要自己显式提供一个 无参构造方法。
    • 带参构造方法可以通过从类的客户端传入参数以初始化实例变量。只有默认值的实例变量往往是不够的，对象也并不总是从初始状态开始创建。
      public class Bird {
    private String featherColor;

    public Bird(String featherColor) {
        this.featherColor = featherColor;
    }

    public void fly() {
        System.out.println("i can fly");
    }
}
      重构后的Bird类添加了带有一个参数的构造方法，传入一个表示羽毛颜色的字符串变量，并用它初始化实例变量featherColor。可能你会产生疑问，这边 实例变量也叫featherColor，形参也叫featherColor。那么怎么区分谁是谁呢。这里我们用了this关键字，this关键字代表当前构造的实例。this.featherColor就是 当前实例的实例变量。现在Bird类的客户端可以用Bird bird = new Bird("red");的方式来创建Bird类的对象。
  • 构造方法的重载
    和普通实例方法一样，构造方法同样可以重载，所谓重载，就是定义多个名称相同，但参数类型或个数不同的方法。
    public class Bird {
    private String featherColor;

    private String birthDay;

    public Bird(String featherColor) {
        this.featherColor = featherColor;
    }

    public Bird(String featherColor, String birthDay) {
        this.featherColor = featherColor;
        this.birthDay = birthDay;
    }

    public void fly() {
        System.out.println("i can fly");
    }
}
    我们添加了一个实例变量记录鸟儿出生日期，然后提供一个带有两个参数的构造函数。

# 类变量和类方法

有时某个状态和行为不属于任何特定的对象，它们是基于类的，我们称其为类变量、类方法

  • 使用static关键字类定义类变量、类方法
  • 示例，Bird类增加了记录天空中有几只鸟的类变量：birdCountInSky及类方法showBirdCountInSky
    public static int birdCountInSky;

static void showBirdCountInSky() {
  System.out.println("天空中有" + birdCountInSky + "只鸟");
}
  • 访问类变量和类方法 直接使用类名.类变量或类名.类方法来访问。如Bird.showBirdCountInSky()

# 访问修饰符与封装性

访问修饰符用来对类，对实例变量和实例方法等进行访问控制。

  • Java中分4种访问控制级别
    • public 对所有类可见（可见的含义是可以使用）
    • protected 对子类可见
    • private 在本类内可见
    • 不加修饰符(友好访问) 包内可见
  • 访问修饰符的作用
    • 有访问修饰符才能谈得上封装性。否则即便纳入包，纳入类，仍不能阻止外部对对象状态的随意更改
    • 访问修饰符的不同级别为面向对象的程序设计提供可能

# 继承

小孩：这只鸟会说话！
他爸：它叫鹦鹉。

鹦鹉也是鸟，会飞，具有鸟所具有的一些特征。但它和其他鸟有所区别的一点是它会学舌。此时，如果我们定义一个鹦鹉类，我们是不是要把鸟的所有 特性再定义一遍呢？所幸，我们不需要这么做，面向对象提供了继承特性，使得我们可以复用已定义特性的同时保留自己的特殊性。

  • 继承使用extends关键字

      class SubClass extends SuperClass {
  }

  • 示例

    public class Bird {
  protected String featherColor;
  protected String birthDay;

  public String getFeatherColor() {
    return featherColor;
  }

  public void setFeatherColor() {
    this.featherColor = featherColor;
  }

  public Bird(String featherColor) {
    this.featherColor = featherColor;
  }

  public Bird(String featherColor, String birthDay) {
    this.featherColor = featherColor;
    this.birthDay = birthDay;
  }

  public void fly() {
    System.out.println("i can fly");
  }
}

public class Parrot extends Bird {

  private String featherColor = "green";
  private String birthDay = "2";

  public Parrot(String featherColor, String birthDay) {
      super(featherColor, birthDay);
  }

  public void showMySuperPower() {
      System.out.println("父类中：");
      super.fly();
      System.out.println("父类中羽毛颜色：" + super.featherColor);
      System.out.println("子类中羽毛颜色：" + this.featherColor);
      System.out.println("父类中出生日期：" + super.birthDay);
      System.out.println("子类中出生日期：" + this.birthDay);
      System.out.println("i am parrot, i can speak, nice?!");
  }
}
    • 调整Bird类的实例变量的访问级别为protected，这样子类可以继承该状态。
    • 定义Parrot类继承（使用extends关键字）Bird类，获得featherColor，birthDay实例变量及fly行为。
    • super调用父类方法，当使用super调用父类构造函数时，super调用必须出现在构造方法中的第一条语句，本例需要显式调用父类构造方法是因为父类并没有提供一个默认构造方法。
    • 子类继承父类，当创建子类的对象时将首先调用父类的构造函数，初始化父类中的状态字段，然后调用本类的构造函数。
    • 当实例化一个子类对象时，实际只会创建子类对象，不会创建父类对象，子类对象具有的父类的状态也会作为子类对象的内容开辟到堆内。
      • 可以通过在IDEA里调试查看
      • 通过字节码分析查看。

  • Java单继承与继承层级
    Java类只能直接继承一个父类extends TheOnlyOneSuperClass。所有类最终都继承自Object。我们想象一个倒挂的树，Object位于树根处，向下依次为子类，子类的子类.... 这样构成了继承层级，越远离Object的层级将越具体。
    Java单继承主要出于Java简单性的设计，既方便编写Java程序的人员，也方便编写Java编译器的人员。

  • 关于继承的使用

    • 不要为了继承而继承（不要仅为了复用代码而继承），这是糟糕的设计，当满足is-a的关系，才可使用继承，否则使用组合。

# 接口

上面介绍了Java是单继承的，但是对于需要用到多继承的场景怎么办？比如有两个用于排序的类如BubbleSorter，QuickSorter，它们都需要一个比较的方法。 假设我们在Comparator类里定义了这个比较的方法，但由于BubbleSorter事先继承了BaseBubbleSorter，QuickSorter事先继承了BaseQuickSorter，它们不能 再继续继承Comparator类，可以使用接口来定义这个比较方法，然后让BubbleSorter和QuickSorter分别实现该接口。

  • 定义

    public interface CompartorI {
    int PUBLIC_STATIC_FINAL_CONST_VAR = 1;
    void compare();
}

public class BubbleSorter implements ComparatorI {
    @Override
    public void compare() {       
    }
}
public interface SubCompratorI extends ComparatorI {
}
    • 使用interface关键字来定义接口，接口之间使用extends关键字继承。
    • 接口只能是包访问级别或公开访问级别，不能用protected或private来修饰，也不能用final修饰，因为定义接口
      就是为了实现的。
    • 接口里可以定义常量，如PUBLIC_STATIC_FINAL_CONST_VAR所示。
    • 接口里的方法是public的，不需要指定修饰符，如果指定其他非public修饰符，将出现编译器错误。
    • 接口里的方法是抽象的（abstract），不能有方法体。Java 8用default修饰的方法可以有具体的实现，但这主要为了在接口里 增加功能而无需实现类修改代码而增加的这个特性。
    • 类使用implements关键字实现接口，实现多个接口时，接口名称用逗号隔开。
    • @Override注解的方法表示实现接口中的方法，如果@Override注解的方法的声明和接口中不一致，那么出现编译错误， 因此使用@Override注解来说明实现接口的意图是良好的实践。

接口可以理解为一组行为规范，实现了接口的类将需要实现接口声明的行为。

# 抽象类

定义

abstract class ClassName {
    实例变量
    实例方法
    抽象方法    
}
  • 如何理解抽象类

类具有一定的抽象，但这是从对现实世界的建模而言来说的。是对具体的对象而言的。但抽象类进一步对软件模型进行抽象，抽象类的继承层级更高。 举个例子，我们对现实建模出Dog和Cat两个类，我们可进一步抽象出动物这个抽象类，比如动物都有出生和死亡及活动这么个行为。

  • 示例
public abstract class Animal {
    protected String birthDay;
    protected BigDecimal age;

    public abstract void live();

    public void simpleDescription() {
        System.out.println("总的来说，我是一只动物");
    }

    public void speak() {
        System.out.println("我不会说人话");
    }
}

public class Dog extends Animal {

    public Dog() {
        this.birthDay = "2021-01-09";
        this.age = new BigDecimal(0.2);
    }

    @Override
    public void live() {
        System.out.println("我为主人看门");
    }

    @Override
    public void speak() {
        System.out.println("我可以汪汪！");
    }
}
  • Animal抽象类的字段从语法上虽可以为private的，但是从语义上来说，抽象类的private字段将毫无用处
  • 抽象类的方法修饰符不能是private和final的。
  • 抽象类的方法可以为抽象方法，可以为带实现的常规方法
  • 实现了抽象的类将继承抽象类所以定义的状态和行为，并实现所有的抽象方法。
  • 如上Dog类的speak方法重写了抽象父类Animal的speak方法，live方法实现了父类的live抽象方法。
  • 不能实例化抽象类，一个动物几乎没有意义。要么是一只猫，一只狗等。
  • 抽象类的作用
    • 实现子类实现代码复用，
    • 提供除使用接口外又一实现多态的机制。

# 多态

多态的特性使得程序的行为在运行期得以确定。编译期无需知道具体的类型，为了阐述这一点，我们继续沿用抽象类一节的例子。我们额外定义一个Cat类

public class Cat extends Animal {
    @Override
    public void live() {
        System.out.println("为主人抓老鼠");
    }
}

public class PolyTest {
    public static void main(String[] args) {
        Animal cat = new Cat();
        live(cat);
        Animal dog = new Dog();
        live(dog);
    }

    private static void live(Animal animal) {
        animal.live();
    }
}

PolyTest的live方法演示了多态行为。方法参数接收一个抽象类型Animal。调用live方法的客户端在调用live时传入Cat还是Dog，决定了live的行为。
