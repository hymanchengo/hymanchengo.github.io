# 线程

任何学习，辅以示例，效果更加。

# 示例一

# 基础示例简介

  • 主线程和MyThread线程共同使用了oo作为锁
  • 若主线程首先占到锁oo,那么主线程首先输出0,...19。
  • 当主线程执行到20时，主线程调用oo.wait，主线程进入等待，让出锁，而且timeout为0，那么等待其他线程的唤醒。
  • MyThread获得oo锁，开始输出0...99
  • 主线程没有任何地方唤醒它，它将永远等待下去。
public class Join {
    public static void main(String[] args) {
        //锁对象资源
        Object oo = new Object();
    
        MyThread thread1 = new MyThread("thread1 -- ");
        thread1.setOo(oo);
        thread1.start();

        synchronized (oo) {
            for (int i = 0; i < 100; i++) {
                if (i == 20) {
                    try {
                        //wait促使当前线程等待
                        //若等待时间为0，则不考虑等待时间，等待其他线程唤醒当前线程
                        //当前线程会释放对oo对象的锁
                        oo.wait(0);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
                System.out.println(Thread.currentThread().getName() + " -- " + i);
            }
        }
    }
    
}

class MyThread extends Thread {
    private String name;
    private Object oo;
    
    public void setOo(Object oo) {
        this.oo = oo;
    }
    
    public MyThread(String name) {
        this.name = name;
    }
    
    @Override
    public void run() {
        synchronized (oo) {
            for (int i = 0; i < 100; i++) {
                System.out.println(name + i);
            }
        }
    }
}

基于基础示例，我们再做些改动，以深入了解。注意：如下的改动都是独立的，测试每个改动都要直接基于基础示例的改动。

# 1. wait

  • 用法obj.wait(),obj.wait(timeout)。
    • timeout为0时相当于wait()。如果没有其他线程通知，那么一直等下去。
  • 使用obj.wait的线程必须持有obj的监视器，即拥有obj锁。wait方法只应该被拥有对象监视器的拥有者线程调用。
  • 虚假唤醒：线程没有从被通知，打断或等待超时的情况下醒来。wait方法应该总是在循环中使用。
      synchronized(obj) {
      while(条件不满足) {
        obj.wait();
      }
      //...
  }
  • 作用是使得使用obj.wait的线程T放入获取obj的等待集合中，放弃该对象的任何锁。T进入休眠状态，不能用于线程调用，除了下面条件满足。
    • 一些其他线程调用了obj.notify或者obj.notifyAll,T线程恰巧被任意选中为唤醒的线程
    • 一些其他线程调用了obj.notifyAll方法
    • 其他线程打断线程T
    • 过了指定等待的时间，如果等待时间为0，则不考虑实际等待时间，线程只是等待被通知。 条件满足后，线程从对象的等待集合中被移除，能重新用于线程调度。它和其他线程一起竞争obj的同步权利。一旦获得了obj的控制，线程里所有 对obj对象的同步将恢复到调用wait前的状态。线程T从wait方法调用中返回。
  • 线程中断
    • 如果当前线程在wait前或wait时被其他任何线程中断。那么将扔出InterruptedException异常，但是直到当前线程获得obj锁状态时才扔出。
    • InterruptedException异常扔出后，线程的interrupted状态将被清空
  • 由于obj.wait是将当前线程加入obj的wait集中，该线程只释放obj锁，对于当前线程占有的其他锁，当前线程在等待obj时并不释放这些对象的锁。

    @Override
    public void run() {

            }
            oo.notify();
        }

# 4. IllegalMonitorStateException异常

  • 修改主线程中synchronize这一行加锁的对象为synchronized (thread1)
  • 主线程扔出异常，因为锁的不是oo，因此尝试使用oo.wait时。抛出IllegalMonitorStateException异常。
  • 本例抛出IllegalMonitorStateException是因为主线程不具有oo监视器，然而却要尝试使用oo来实现等待。

@Override
